When a dynamic video's background URL can't be reached, Sendspark handles it in two ways. If a lead has no website URL at all, the system uses a fallback background so the video still generates. If a URL is provided but fails to load, that video is rejected — you can fix the URL in the Contacts section of your campaign and resubmit.

## Key takeaways

- **No website URL provided:** Sendspark automatically uses a fallback background so the video still generates. Enable it by setting a Fallback URL, then at the Contacts step click the settings wheel and select **Generate Video Using Fallback Data** in the Error Handling panel.
- **Invalid or unloadable URL:** if a URL is provided but fails to load or render, the video generation is rejected. Review rejected leads in the **Contacts** section, edit the URL that failed, and resubmit.
- The Error Handling panel offers two choices: generate the video using fallback data, or do not generate it, show an error, and let you update the information.
- Videos generated from fallback content are easy to spot afterward — they show an orange status indicator and an orange play button.

## Learn how to handle background URLs the right way:

## There are a few different scenarios that can happen whenever you are rendering videos in Sendspark.

When using dynamic video campaigns, here’s how the system handles leads based on their website URL status:

- **No Website URL Provided:** If a lead does **not** include a website URL (i.e., the URL field is left completely empty), the system will automatically use a **fallback background** for the video. This ensures that the video is still generated and usable. To activate this option, simply create a new Dynamic Video Campaign and follow these steps: 1) Set up the Fallback URL you want to use when your lead has no data included: <img src="/images/11178712/e4bdeb76-image.png" alt="Dynamic Video Campaign setup form with Fallback URL field highlighted" /> 2) Once you reach the 3rd step: Contacts, you'll need to click on the Settings wheel on the right-hand side to display the ERROR HANDLING panel. From there, make sure the first option is selected: **Generate Video Using Fallback Data:**

![Contacts step settings panel with Generate Video Using Fallback Data option selected](/images/11178712/a3a707e4-image.png)

And that's it! The system will automatically complete the video rendering process using the most suitable parameters available. Additional tips and insights for managing missing fields are provided below.

- **Invalid or Unloadable Website URL:** If a website URL is provided but **fails to load or render properly**, the video generation will be **rejected**. <img src="/images/11178712/978de147-image.png" alt="Error message indicating invalid or unloadable website URL rejection notification" /> You can review these rejected leads in the **Contacts** section of your dynamic video campaign: <img src="/images/11178712/5dbf75f1-image.png" alt="Contacts section of dynamic video campaign showing rejected leads list" /> Once in **Contacts**, edit the URL that failed: <img src="/images/11178712/408f5c05-image.png" alt="Contact edit form with URL field ready for correction and resubmission" />

And that's it! Let us know if you have any questions :)
