Fallback websites are default landing pages that Sendspark shows as a dynamic-video background when a prospect's own site can't be displayed — for example, an API failure, site downtime, or security blocks. Set one during campaign setup and leave the website property field empty to apply it, or turn the option off in account settings so the video fails instead.

## Key takeaways

- A fallback website is a default background used when the prospect's site can't be displayed, keeping the video professional instead of failing.
- Fallbacks appear when Sendspark can't reach the site: an unanswered API call, website downtime, or security protocols (such as banks) that block API requests.
- Set the fallback during campaign setup and leave the **Website** property field empty so Sendspark uses it for all videos without a specified site.
- You can disable fallbacks in your account's dynamic video preferences so the video fails instead of showing a fallback background.

## How Do Fallback Websites Work in Sendspark's Dynamic Video Feature?

Dynamic videos in Sendspark leverage fallback websites to ensure a seamless experience when a prospect's site cannot be displayed. This guide outlines the purpose of fallback websites, the setup process, common issues, and how to configure your settings.

## What Are Fallback Websites?

Fallback websites act as default landing pages that appear when the intended prospect's site cannot be displayed in dynamic videos. They are crucial in maintaining usability and a professional appearance during any playback issues.

## How to Set Up a Fallback Website in HubSpot Workflows

To ensure a fallback website appears in HubSpot workflows in place of a prospect's site, follow these steps:

1. **Begin Campaign Setup:** Navigate to your Sendspark campaign settings.
2. **Select the Fallback Website:** During the initial configuration steps, choose the fallback website that should be used.
3. **Leave the Website Property Field Empty:** By doing so, Sendspark will automatically use the configured fallback website for all videos where no prospect site is specified.

## Why Does Sendspark Display Fallback Websites in Some Cases?

When generating dynamic videos, Sendspark attempts to use the target prospect's website as a background. However, fallback websites will appear under these circumstances:

- **API Issues:** Sendspark cannot reach the website due to an unanswered API call.
- **Website Downtime:** The site is temporarily down, preventing content retrieval.
- **Security Protocols:** Some websites, such as those hosted by banks or secured with additional layers of protection, may block API requests.

In such cases, fallback settings ensure that videos continue to display a functional background rather than failing entirely.

## How to Adjust Fallback Settings

You can control how fallback websites function by adjusting account configurations. To disable fallback websites and allow the video to fail instead when the prospect's site cannot be used:

1. Navigate to your Sendspark account settings.
2. Locate the fallback option in the dynamic video preferences.
3. Turn off the fallback option to prevent a fallback website from being displayed.

## Conclusion

Fallback websites help maintain a professional and seamless dynamic video experience. Whether setting up workflows or troubleshooting API failures, understanding fallback behavior equips you with the tools to optimize your campaigns effectively.
