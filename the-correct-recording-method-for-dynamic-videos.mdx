To record correctly for a Dynamic Video, set the recording method to **"Face"** in the Sendspark Chrome extension. The default mode is **"Both"** (face plus screen), which is great for standard videos but produces an unwanted result in Dynamic Videos. Switch to Face using the icons at the bottom of the recorder before you record.

## Key takeaways

- Use the **"Face"** recording method for Dynamic Videos, set from the bottom icons in the Sendspark Chrome extension.
- The default method is **"Both"** (records your face along with your website/screen); the active mode appears darker than the others.
- "Both" is the best mode for regular videos, but recording in "Both" and using it in a Dynamic Video gives an unwanted result.
- Set the mode to "Face" *before* recording so your Dynamic Video displays as intended.

The first step towards recording your first videos is to click on the Sendspark Chrome extension icon. Doing that will open your camera's feed in a bubble on top of your background, just like this:

![Sendspark Chrome extension camera bubble overlay displaying user face on website](/images/9565499/9a542bc6-image.png)

This recording method is set by default and it is the best way to record yourself along with your website. It is called **"Both"**and it can be changed from the bottom icons:

![Sendspark recording mode icons at bottom showing Both mode as active selection](/images/9565499/8c88842f-image.png)

You'll notice that the option is darker than the rest, showing us that it is an active selection.

If you want to correctly record a Dynamic Video, the option needs to be set to **"Face".**

Recording a video in "Both" and using it in a Dynamic Video will provide an unwanted result:

![Dynamic Video example showing unwanted result from recording in Both mode](/images/9565499/06d8a3c4-image.png)

## Why not use "Both" mode for Dynamic Videos?

"Both" mode records your face and your screen at the same time, which is ideal for standard videos. But Dynamic Videos are designed to place just your face over a personalized background. When a "Both" recording is used in a Dynamic Video, the result is not what you want. Recording in "Face" keeps only your face so it can sit correctly on the dynamic background. For a deeper comparison, see [Face Recording vs Both Recording in Dynamic Videos](/face-recording-vs-both-recording-in-dynamic-videos).
