Sendspark can't automatically pull a prospect's website from their email address while skipping free domains like Gmail, but you can build this in HubSpot. Create a Background URL contact property, then a HubSpot workflow that sets it to the prospect's website when known and to your website when it isn't. Use that property when importing HubSpot leads into Sendspark.

## Key takeaways

- Sendspark does not natively derive a website from an email address while excluding free domains like Gmail; you build the logic in HubSpot instead.
- Create a new **Background URL** contact property in HubSpot.
- Use a HubSpot workflow to set Background URL to the prospect's website when known (auto-populated for company email addresses) and to your website when it isn't.
- Map that Background URL property when importing HubSpot leads into Sendspark to make dynamic videos.

You might be wondering... "Is it possible for Sendspark to show the prospects website from their email address, unless it's a free email address like Gmail?"

And while Sendspark does not have this advanced functionality, you can do this yourself in HubSpot by creating a new "Background Url" contact property.

You would simply...

- Create a new Contact Property in HubSpot: **Background URL**
- Create a HubSpot Workflow to determine that Background URL should be for each contact. Likely...
  - When their website URL is known (which would happen automatically in HubSpot when they have a company email address): **their website URL**
  - When their website URL is not know by HubSpot: **your website URL**

Then, you can use this property, "Background URL" whenever you are importing HubSpot leads into Sendspark to make dynamic videos.
